Microsoft 365 OAuth Instructies

Om e-mail te versturen via Microsoft 365 met OAuth2 authenticatie heeft u de volgende gegevens nodig uit de Azure Portal:

  1. Client ID - De applicatie ID van uw geregistreerde app in Azure AD
  2. Tenant ID - De directory (tenant) ID van uw Azure AD tenant
  3. Client Secret - De Value (niet de ID!) van een client secret die u heeft aangemaakt
  4. Hoofdgebruiker E-mailadres - Het e-mailadres dat u wilt gebruiken voor authenticatie

Let op: Bij het invoeren van de Client Secret, zorg ervoor dat u de volledige waarde (Value) kopieert en niet de ID. De waarde is meestal langer en complexer dan de ID.

Belangrijke opmerking over Client Secret:

Als u de foutmelding AADSTS7000215: Invalid client secret provided krijgt, controleer dan of u de juiste waarde heeft ingevoerd. In Azure Portal moet u de Value (waarde) gebruiken van het client secret, niet de ID. Deze waarde is alleen zichtbaar direct na het aanmaken van de secret.

Stappen om een app te registreren in Azure:
  1. Ga naar https://portal.azure.com
  2. Navigeer naar Azure Active Directory > App registrations
  3. Klik op "New registration"
  4. Kies een naam, en stel http://localhost in als redirect URI (voor desktop)
  5. Na registratie, noteer de Application (client) ID en Directory (tenant) ID
  6. Ga naar Certificates & secrets, en voeg een Client Secret toe
  7. Onder API Permissions: voeg toe SMTP.Send via Microsoft Graph en klik op "Grant admin consent"

Microsoft 365 OAuth 2.0 E-mail Integratie

OAuth 2.0 biedt een veiligere manier van authenticatie dan traditionele wachtwoorden. Het stelt de applicatie in staat om e-mails te versturen namens een gebruiker zonder dat het wachtwoord bekend hoeft te zijn.

Voordelen van Microsoft 365 OAuth
  • Verbeterde beveiliging

    Geen wachtwoorden opgeslagen, alleen tokens met beperkte levensduur

  • Multi-factor authenticatie ondersteuning

    Werkt naadloos met MFA-instellingen van uw Microsoft 365 account

  • Granulaire permissies

    Alleen specifieke rechten worden verleend, zoals alleen e-mail verzenden

  • Automatische token vernieuwing

    Tokens worden automatisch vernieuwd zonder handmatige interventie
